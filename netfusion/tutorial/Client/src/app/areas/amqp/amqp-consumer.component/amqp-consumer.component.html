Overview

  • Built using the AmqpNetLite open-source library.
  • Extends the existing method attributes used to identity message handlers.
  • When a message handler is decorated with the Queue or Topic message handler attributes, the method is added as a subscription to the corresponding Queue or Topic.
  • When a message is received, the message handler is invoked.

Usage Summary

  • Create class representing the command/domain-event to be received.
  • Define message handler decorated with the Queue/Topic attributes specifying the broker queue/topic to be subscribed.
  • Decorate the handler class with the Host attribute specifying the host name as configured within the application settings.

Subscribing to Messages

The following corresponds to the code contained within Service.Client host responding to Messages as they arrive on the defined queue and topic. Before message handlers can be subscribed to queues and topics, the application must have the connection information specified within appsettings.json.

  • Device message handler class implementing the IMessageConsumer marker interface.
  • Decorate the handler class with the Host attribute indicating the name of the host as configured within the application's settings.

Subscribing Command to Queue

  • Device command having the structure matching the message that will be sent and serialized to the queue.
  • Define message handler method taking as the parameter of the command type to be handled.
  • Decorate the message handler method with the Queue attribute specifying the name of the queue to which the handler method should subscribe.

Subscribing Domain-Event to Topic

  • Device domain-event having the structure matching the message that will be sent and serialized to the topic.
  • Define message handler method taking as the parameter of the domain-event type to be handled.
  • Decorate the message handler method with the Topic attribute specifying the name of the topic and subscription to which the handler method should subscribe.
