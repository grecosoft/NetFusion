Overview

  • Extends .net Core configuration.
  • Allows for configuration classes to be injected directly into dependent components.
  • Supports validation of loaded configuration values.

Usage Summary

  • Configuration class implements the IAppSettings interface.
  • Configuration class decorated with the ConfigurationSection attribute.
  • The ConfigurationSection specifies the path to the section in the configuration.
  • The path is a colon separate path.
  • A configuration can be validated when created for the first type by implementing the IValidatableType interface.
  • Instance of the settings class can be injected into a dependent component.

Suggestions

  • If the settings is domain-specific, it can be placed in the solution's domain project.
  • If the settings are for configuring a specific technology implementation, it can be placed in the solution's infra project.
