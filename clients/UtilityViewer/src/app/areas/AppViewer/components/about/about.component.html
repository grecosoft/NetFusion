Application Overview Allows viewing how a NetFusion based application was composed from plug-in components. A composite application consists of the following types of plug-ins:
Host
The assembly representing the executable process. This will most often be a ASP.NET Core WebApi project. There can only be one Host plug-in assembly.
Application
Plug-in components containing logic pertaining to the business application solution. There will usually be application plugins for the following: Domain, Api, Application, and Infrastructure
Core
Plug-in components containing cross-cutting implementation specific components. Theses type of plug-ins are not specific to any given business application and can be optionally shared across applications.
A plug-in invokes logic during the boot strap process by defining one or more PluginModule derived classes. These derived class instances have their methods invoked at specific times during the boostrap process.
